Using Jenkins to trigger mvn tasks

This is the most common way that you will use mvn to interact with your sandbox environment. There are a number of jenkins jobs already populated in the sandbox and they can be copied and modified to do most routine build, test and deploy tasks.

Using command line mvn

In general sandboxes don't come with a shell account so running mvn directly on the command line on the sandbox will be uncommon. However, if you are set up to do CQ deploys using mvn from your own machine already, most of those tasks can take a hostname argument that wil lallow them to deploy directly to the sandbox.
